Under the Hood

Why this isn't just a chatbot on your spreadsheet.

For the technical reviewers. Every query through Kratu runs a precision retrieval pipeline — not a keyword search, not a simple prompt.

🧠
RAG + GraphRAG ChromaDB semantic search finds relevant records. GraphRAG understands relationships between students, courses, and regulations.
🎯
CrossEncoder Reranking After retrieval, results are re-scored by a CrossEncoder model. The difference between a close answer and the correct answer.
🔒
Firebase Auth + RBAC A student's query only retrieves their own records. Faculty see their department. HODs see everything. Enforced at the API layer.
📊
Observability Dashboard Every query is logged with latency, confidence score, source citations, and user role. You see what your college is asking Kratu.
query_pipeline.py
01→Auth check (role: student)12ms
02→Embed query (SBERT)43ms
03→ChromaDB retrieval (k=15)118ms
04→CrossEncoder rerank67ms
05→LLM generation + cite540ms
06→Log to Langfuse9ms
Total: 0.9s · Confidence: 94%
